# 开发备忘：授权构建与发布流程

## 一、授权体系架构

```
┌─ D:\AIkey\SECURITY_KEY.dat ──────────────────────┐
│  64 字节随机密钥（唯一，丢失无法恢复）              │
│  授权签发和验证共用此密钥                          │
└───────────────────────────────────────────────────┘
                         │
          build_embed_key.py 读取（构建时）
                         ▼
┌─ _secret.py（临时生成，已 .gitignore）──────────┐
│  EMBEDDED_KEY = base64(SECURITY_KEY.dat)        │
│  PyInstaller 编译为 .pyc 打包进 exe 内部         │
└──────────────────────────────────────────────────┘
                         │
                 exe 运行时载入
                         ▼
┌─ license_check.py ──────────────────────────────┐
│  ① 从 _secret 模块加载内嵌密钥                   │
│  ② 读取 exe 同目录 license.dat                   │
│  ③ 校验 HMAC-SHA256 签名                        │
│  ④ 校验机器指纹（CPU+主板+磁盘+MAC）             │
│  ⑤ 校验过期时间                                  │
│  ⑥ 通过 → 启动；失败 → 退出                      │
└──────────────────────────────────────────────────┘
```

## 二、文件说明

| 文件 | 作用 | 是否入库 |
|------|------|----------|
| `license_check.py` | 授权验证核心模块（复制自 D:\AIkey） | ✅ |
| `build_embed_key.py` | 构建时从 D:\AIkey\SECURITY_KEY.dat 生成 _secret.py | ✅ |
| `_secret.py` | 内嵌密钥（临时文件，构建时自动生成） | ❌ .gitignore |
| `build.bat` | 一键构建脚本 | ✅ |
| `build.spec` | PyInstaller 打包配置（onefile 模式） | ✅ |
| `post_build.bat` | 构建后处理（复制 Chromium 等） | ✅ |
| `SECURITY_KEY.dat` | 密钥文件（仅存于 D:\AIkey） | ❌ 不入库 |
| `license.dat` | 授权文件（发给用户） | ❌ 不入库 |

## 三、构建 exe 流程

### 3.1 前置条件

- conda/Python 环境已安装所有依赖（`requirements.txt`）
- `D:\AIkey\SECURITY_KEY.dat` 密钥文件存在
- Playwright Chromium 已安装（`python -m playwright install chromium`）

### 3.2 执行构建

```bash
# 在项目根目录运行
build.bat
```

批处理自动完成三步：
1. `python build_embed_key.py` → 生成 `_secret.py`（从 D:\AIkey\SECURITY_KEY.dat 读取密钥）
2. `pyinstaller build.spec` → 打包为 onefile 模式的 WeChatOA.exe
3. `post_build.bat` → 复制 Chromium 浏览器、.env 到 dist/ 目录

### 3.3 产物

```
dist/
├── WeChatOA.exe         ← 主程序（~109 MB，含全部 Python 依赖和内嵌密钥）
├── chrome-win/          ← Playwright Chromium 浏览器（外置）
└── .env                 ← 环境配置模板
```

## 四、授权工作流

### 4.1 首次给用户授权

1. **获取用户的机器码**：用户在 exe 同目录放一个空的 `get_machine_code.txt` 之类的（或报错时显示）
2. **生成授权文件**：

```bash
cd D:\AIkey\license_system
python generate_license_tool.py
# 输入用户名、机器码、有效期天数
# 输出 license.dat
```

3. 将 `license.dat` 发给用户，放到 `WeChatOA.exe` 同目录

### 4.2 用户首次使用

```
解压包目录/
├── WeChatOA.exe
├── chrome-win/          ← 浏览器引擎
├── .env                 ← 用户编辑配置（API Key 等）
└── license.dat          ← 你发给用户的授权文件
```

用户直接运行 `WeChatOA.exe`，授权通过后自动启动 Web 服务。

### 4.3 版本升级

1. 你修改代码 → 运行 `build.bat` 
2. 将新版的 `WeChatOA.exe` 发布到 GitHub Releases
3. **用户操作**：下载新 exe，覆盖旧文件
4. `license.dat`、`chrome-win/`、`.env` 全部保留，无需重新授权

## 五、GitHub 发布流程

### 5.1 日常提交代码

```bash
git add -A
git commit -m "描述本次修改"
git push
```

### 5.2 创建 Release（给用户下载）

1. 浏览器打开仓库：`https://github.com/cheng0009/WeChat-OA-Assistant`
2. 点击右侧 **Releases** → **Create a new release**
3. 填写版本号（如 `v1.1.0`）和更新说明
4. 将 `dist/WeChatOA.exe` 拖拽上传作为附件
5. 点击 **Publish release**

### 5.3 新用户首次安装包

如果需要给新用户完整的首次安装包，需要包含：

```
WeChat-OA-Assistant-v1.0.zip
├── WeChatOA.exe
├── chrome-win/                    ← 复制自 %USERPROFILE%\AppData\Local\ms-playwright\chromium-1169\chrome-win
└── .env                           ← 用户自行编辑
```

## 六、注意事项

### 6.1 密钥安全（最重要）

- `D:\AIkey\SECURITY_KEY.dat` 是唯一密钥，**必须备份**
- 备份到至少两个安全位置（如 U 盘、云盘加密存储）
- 密钥丢失 → 所有已发出的授权文件全部作废，不可恢复
- 密钥泄露 → 他人可伪造授权文件

### 6.2 构建环境

- 当前构建环境：conda Python 3.13（`F:\livehuman_DY1113\miniconda3`）
- `wechatsogou` 与新版 Werkzeug 不兼容，已手动打补丁到 `wechatsogou\filecache.py`
- 如果更换构建机器，需重新安装依赖并打补丁

### 6.3 授权限制

- 免费版/测试版：可不绑定机器码，设 `max_devices=3`，供有限使用
- 付费版：绑定机器码，设 `max_devices=1`，限定单台设备
- 授权到期后需重新生成 `license.dat`

### 6.4 Chromium 兼容性

- `post_build.bat` 中 Chromium 路径写死了 `chromium-1169`
- 如果 Playwright 更新导致 Chromium 版本号变化，需修改 `post_build.bat` 中的路径
- 或者改用 `python -m playwright install chromium` 后自动检测路径
